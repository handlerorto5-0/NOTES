Concurrent Mode
В параллельном режиме React может планировать задачи более детально, приостанавливая свои рабочие элементы, проверяя различия и обновляя DOM для предыдущих изменений состояния, чтобы убедиться, что он реагирует на действия пользователя.
Параллельный режим включает несколько новых хуков, useTransition и useDeferredValue, которые улучшают взаимодействие с пользователем, сглаживая переход от одного представления к другому или от одного состояния к другому.

Suspense
Компоненты, созданные для работы с приостановкой, теперь могут приостанавливаться, если они не готовы вернуть свой пользовательский интерфейс. Они могут ждать загрузки кода компонента, ресурсов или данных и просто еще не имеют информации, необходимой для полного описания своего пользовательского интерфейса. React может приостановить обработку приостановленного компонента и продолжить обход дерева элементов.
React предоставляет компонент Suspense, который можно использовать для закрытия дыр, оставленных приостановленными компонентами в вашем пользовательском интерфейсе.

<Suspense fallback={<MySpinner />}>
<MyFirstComponent />
<MySecondComponent />
</Suspense>
