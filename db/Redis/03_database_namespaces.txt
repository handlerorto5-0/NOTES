Пространства имен базы данных

До сих пор мы взаимодействовали только с одним пространством имен. Иногда нам нужно разделить ключи на несколько пространств имен. Например, если вы написали интернационализированное хранилище ключей и значений, вы можете хранить разные переведенные ответы в разных пространствах имен. Ключевое приветствие может быть установлено на «guten Tag» в немецком пространстве имен и «bonjour» на французском. Когда пользователь выбирает свой язык, приложение просто извлекает все значения из назначенного пространства имен.
В номенклатуре Redis пространство имен называется базой данных и имеет номер. До сих пор мы всегда взаимодействовали с пространством имен по умолчанию 0 (также известным как база данных 0). Здесь мы устанавливаем приветствие на английский hello.

> SET greeting hello
> GET greeting

Но если мы переключимся на другую базу данных с помощью команды SELECT, этот ключ будет недоступен.
> SELECT 1
> GET greeting  вернёт nil
Можно добавить туже переменную с другим значением
> SET greeting "guten Tag"

Поскольку все базы данных работают на одном экземпляре сервера, Redis позволяет нам перемещать ключи в случайном порядке с помощью команды MOVE. Здесь мы перемещаем приветствие в базу данных 2:
> SELECT 0
> MOVE greeting 2
> SELECT 2
> GET greeting
